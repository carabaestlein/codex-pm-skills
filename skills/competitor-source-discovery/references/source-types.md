# Source Types

Use this priority order when assembling competitor monitoring sources.

## Primary Sources

- **Blog**: Product, company, engineering, or customer blog.
- **Changelog**: Changelog, release notes, product updates, what's new.
- **Docs**: Product documentation, API docs, admin docs, help center.
- **Pricing**: Pricing page, plans page, packaging page, quote/contact sales page when public pricing is absent.
- **Integrations**: App marketplace, integrations directory, partner directory.
- **Trust**: Security, compliance, trust center, privacy, subprocessors, status page.
- **Press**: Press room, news page, announcements page.
- **Events/Webinars**: Public events, webinars, launch sessions.

## Secondary Sources

Use secondary sources when they add useful perspective or when primary sources are thin.

- Review profiles: G2, Capterra, Gartner Peer Insights, TrustRadius.
- Marketplaces: Salesforce AppExchange, HubSpot Marketplace, AWS Marketplace, Chrome Web Store, Slack App Directory.
- Social/product communities: LinkedIn company posts, Product Hunt, community forums.
- Jobs/careers: Useful for weak hiring or strategy signals; label as secondary.

## Exclusions

Avoid:

- Generic homepages when a more specific source exists.
- Undated marketing pages unless they are central to pricing, packaging, or positioning.
- Third-party listicles, SEO pages, or scraped directories.
- Private or login-required URLs.
- Regional pages unless the workflow is explicitly region-specific.
