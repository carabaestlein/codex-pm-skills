# Output Formats

Use only the formats requested in the config or user prompt. If none are specified, provide the default set.

## Release Note

```markdown
## <Feature name>

<1-2 sentence factual summary of what changed and who it helps.>

- <Specific capability or behavior>
- <Specific capability or behavior>
- <Availability, prerequisite, or limit if relevant>
```

## Changelog Entry

```markdown
- Added <feature/capability> so <user/persona> can <specific job or workflow>. <Availability or setup note if relevant.>
```

## LinkedIn Post

```markdown
<Short hook grounded in the user problem.>

<What changed in plain language.>

<Who it is for and when it is useful.>

<CTA or docs link if provided.>
```

## Short Social Post

```markdown
<One or two short sentences about the feature and user-visible change.>
```

## Internal Enablement Blurb

```markdown
**What changed:** <factual summary>
**Who cares:** <ICP/persona>
**How to position it:** <concise positioning guidance>
**Do not claim:** <known constraints or risky claims>
```

## Open Questions

Use this section when facts are missing:

```markdown
## Open Questions

- <Question needed to avoid an inaccurate claim>
```
